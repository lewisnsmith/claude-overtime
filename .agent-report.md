# Wave 3 Agent C2: E2E Test Harness

All 8 scenarios pass (70/70 assertions) on a clean run of `bash tests/e2e/run-all.sh`.

## Partially automated scenarios

- **single-task-happy-path**: exercises state lifecycle (create/show/reset/restore) and settings backup/restore but does NOT invoke a real Claude session or `caffeinate` sleep — that portion requires a live claude-code runtime and is impractical to automate in a bash harness.
- **backlog-happy-path**: similarly tests state + cursor serialization but cannot exercise actual parallel worktree agent spawning (would require a live Claude Code session and real git remotes).

## Notes

- Scenario 7 (`mechanical-safety`) uses `cd "$TMP_PROJECT"` inside a subshell so the hook's `$PWD`-based project-root check works correctly — setting `PWD=` via env alone is insufficient.
- Scenario 6 (`crash-safety`) skips gracefully if PID 99999 happens to be alive on the test machine.
- All mock scripts log to `$MOCK_LOG` for post-hoc inspection.
